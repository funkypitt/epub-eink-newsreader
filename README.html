Magazine Reader (debug)

Magazine-mode ePub reader, fork of Book's Story.

Overview

Fork of Book's Story (ua.acclorite.book_story) with an opinionated magazine-mode parser and TOC for ePub issues produced by an in-house newsletter generator (calibrated against NZZ, 24 heures, Courrier International, and The Economist fixtures).

The fork detects magazine ePubs at open time, paginates the table of contents with one card per article (category, title, optional cover image), and routes article taps to a per-article view rendered by an in-zip WebView. A permanent three-zone navigation header (prev / home / next) sits on top of the existing tap-zone gesture system on every magazine screen.

Debug-signed, sideload-only. The .debug application id suffix lets it coexist with the upstream Book's Story release on the same device. See MAGAZINE-READER-phases-1-2.md for the parser and TOC spec.

Features

  • All upstream Book's Story features (Material 3 reading, Jetpack Compose UI, library, customisation)
  • Magazine-mode parser detecting in-house newsletter ePubs at open time
  • Paginated TOC with one card per article (category, title, optional cover image)
  • Per-article WebView rendering of the in-zip XHTML
  • Permanent three-zone navigation header (prev / home / next) on magazine screens
  • Debug-signed; coexists with upstream Book's Story (.debug applicationId suffix)

Build & install

Standard Android Gradle (Kotlin DSL) project; min SDK 26, target SDK 36.

./gradlew :app:assembleDebug
# APK at app/build/outputs/apk/debug/

The release-debug variant builds the release pipeline (R8 / minification) but signs with the debug key, useful for local sanity-checking.
